# Git Commit Convention

Git Commit 규칙
(커밋하기 전에 한 번씩 확인하기🙏)

| Type | 설명 |
|------|------|
| feat | 새로운 기능 |
| fix | 버그 수정 |
| style | 스타일(CSS/UI) 수정 |
| refactor | 코드 구조 개선 |
| chore | 설정/패키지/배포 |
| docs | 문서 수정 |
| perf | 성능 개선 |
| test | 테스트 |

## Examples

```text
feat: Hero 섹션 구현
fix: 모바일 메뉴 오류 수정
style: 레이아웃 수정
refactor: 컴포넌트 분리
chore: pnpm 적용
docs: README 작성
